build1: here, i gave the AI a permanent past so it doesn't forget everything when the terminal closes. first, i used the uuid library to generate a unique 8-character ID for the session. then I implemented json.dump() to bundle the ID, title, time, and the whole messages list into a single JSON file on my hard drive. to recall the past, i used json.load() to hunt the file by its ID and hand the dictionary back to python. i also wrote a function to scan the sessions folder and sort the metadata so the newest chats are at the top. lastly, instead of a hardcoded system prompt, i made the AI read its rules from an external AGENTS.md file and glue it to the base prompt. this way i can change the AI's behaviours instantly without touching the python code.

build2: here,I learned how to completely separate the AI's brain from its body. instead of having the terminal UI and chat loops tangled up with the api calls,I built a silent base Agent class. this class acts as the pure brain—it handles the ReAct loop, tool dispatching, and json memory saving entirely in the background. then, I created a tiny REPLAgent to act as the terminal interface that just inherits the brain. this means I can run the exact same AI logic in a terminal, a single cli command, or a full UI later.

I also gave the AI massive upgrades to its hands with opencode-style file tools. instead of dumping a whole file into memory and crashing the API like we did before, I implemented an advanced read_file that reads specific line numbers. i also built an edit_file tool so the AI can replace, delete, or insert specific lines of code in existing files. finally,I added a resolve_path security guard to ensure the AI can never accidentally read or write files outside of my project workspace.


agent setup & cli routing: here, I pulled everything together into one mastermind agent.py script. i understood how to use sys.argv to read command-line arguments and route the user to different modes. for example, if i type --session [id], the code intercepts the id and injects the past json memory so i can pick up where i left off. i also learned the concept of "lazy importing"—i only import the TUIAgent library inside the --tui if-statement. this way, if i just want to run a quick terminal command, it skips loading the massive textual library and runs lightning fast.

web tools: i basically reused the logic i learned in week 2 for google search and web scraping, but properly modularized them inside tools/web.py. i understand how to use the requests library with my serper key to get search results, and trafilatura to extract the main text from a url. one important thing i noted and implemented was truncating the webpage text using content[:12000]. this safely protects the AI's context limit so dumping massive websites into its memory doesn't crash the api.

paper tools: this replaced the fake alphaXiv server from week 2 with a real academic database. i wrote functions to connect directly to the hugging face papers api. i learned the syntax of querying their search endpoint (/api/papers) to get a list of papers, and then making a second request to /api/papers/{arxiv_id}.md to get the actual markdown text of the paper. i also added a logical fallback so if the server returns an error for the full text, it automatically just gives the AI the paper's abstract instead.

tui and final wiring: this was where i snapped the face onto the silent ai brain. i created a TUIAgent class that inherits the base Agent, but i overrode the _emit() function to beam the tool logs (like "Calling tool: search_web") into a textual UI panel. just like week 2, i used the concept of Asynchronous Programming with @work(thread=True) so the AI's api calls happen in the background and the chat window doesn't freeze or lock up the user's keyboard. then i used call_from_thread() to safely push the ai's final response back onto the main screen.
